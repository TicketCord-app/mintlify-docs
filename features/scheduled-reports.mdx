Get regular reports delivered to your inbox with key support metrics and insights.

<Info>
  Scheduled reports are an **Enterprise** feature.
</Info>

## What Are Scheduled Reports?

Scheduled reports automatically send email summaries of your support performance at regular intervals. They include:

- Ticket volume and trends
- Resolution times
- Staff performance
- SLA compliance
- AI feature usage

## Setting Up Reports

<Steps>
  <Step title="Navigate to Reports">
    Dashboard → Analytics → Scheduled Reports
  </Step>
  <Step title="Click Create Report">
    Select **Create New Schedule**
  </Step>
  <Step title="Configure Content">
    Choose what metrics to include
  </Step>
  <Step title="Set Schedule">
    Define frequency and timing
  </Step>
  <Step title="Add Recipients">
    Enter email addresses
  </Step>
  <Step title="Save Schedule">
    Report schedule is now active
  </Step>
</Steps>

## Report Content

### Available Metrics

Choose which sections to include:

| Section | Contents |
|---------|----------|
| **Summary** | Key numbers at a glance |
| **Ticket Volume** | New, closed, open tickets |
| **Response Times** | First response averages |
| **Resolution** | Time to close metrics |
| **Staff Performance** | Individual metrics |
| **SLA Compliance** | Target achievement rates |
| **Categories** | Breakdown by ticket type |
| **AI Usage** | Token consumption, KB stats |

### Customizing Content

For each section, you can:

- Include or exclude
- Set comparison period (vs last period)
- Add specific metrics

## Report Frequency

### Schedule Options

| Frequency | Delivery |
|-----------|----------|
| **Daily** | Every day at specified time |
| **Weekly** | Specific day of week |
| **Monthly** | Specific date each month |
| **Custom** | Define your own schedule |

### Timing

- Set delivery time (e.g., 8:00 AM)
- Choose timezone
- Reports generated and sent at scheduled time

### Example Schedules

| Name | Frequency | Time |
|------|-----------|------|
| Daily Summary | Daily | 8:00 AM |
| Weekly Review | Monday | 9:00 AM |
| Monthly Report | 1st of month | 6:00 AM |

## Recipients

### Adding Recipients

Enter email addresses for people who should receive reports:

- Team members
- Managers
- External stakeholders

### Multiple Recipients

Each report can have multiple recipients:

- All receive the same report
- All receive at the same time
- Individual unsubscribe links

### External Recipients

You can add non-TicketCord users:

- They receive email but can't access dashboard
- Good for managers who don't need full access
- Must have valid email address

## Report Format

### Email Layout

Reports arrive as formatted emails with:

- Clean, professional design
- Charts and graphs (as images)
- Summary tables
- Links to detailed dashboards

### PDF Attachment

Optionally include:

- PDF version attached
- Good for archiving
- Easy to forward

## Managing Schedules

### Viewing Schedules

See all your scheduled reports:

| Info | Description |
|------|-------------|
| **Name** | Report identifier |
| **Frequency** | How often it runs |
| **Next Run** | When it will run next |
| **Recipients** | Who receives it |
| **Status** | Active/Paused |

### Editing Schedules

1. Click on the schedule
2. Modify settings
3. Save changes

### Pausing Schedules

Temporarily stop reports:

1. Toggle status to Paused
2. Reports won't send
3. Toggle back to resume

### Deleting Schedules

1. Click delete icon
2. Confirm deletion
3. Schedule is removed

## Report History

### Viewing Past Reports

Access previously sent reports:

- Date sent
- Recipients
- Download report
- Resend option

### Retention

- Reports stored for 90 days
- Download before expiration
- Export for long-term storage

## Data Scope

### Per-Server Reports

Reports cover data for:

- Selected Discord server
- Selected time period
- Selected metrics

### Multi-Server Reports

Enterprise can create reports spanning:

- All servers
- Selected servers
- Aggregated or separate

## Best Practices

<Tip>
  **Match Frequency to Need**
  Daily reports for active teams, weekly for oversight.
</Tip>

<Tip>
  **Keep It Focused**
  Include relevant metrics only. Too much data is overwhelming.
</Tip>

<Tip>
  **Right Time of Day**
  Send before start of workday for best visibility.
</Tip>

<Tip>
  **Review Recipients**
  Periodically verify list is current.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Report not arriving">
    Check spam folder. Verify email address is correct.
  </Accordion>

  <Accordion title="Data looks wrong">
    Confirm timezone settings and date range.
  </Accordion>

  <Accordion title="Charts not displaying">
    Some email clients block images. Download PDF instead.
  </Accordion>
</AccordionGroup>
